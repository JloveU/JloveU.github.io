本文记录本人在编程过程中总结的、供自己参考的一些编程规范（主要是C和C++）。
其中也包括很多从书籍或者网上获得的规范、技巧等。

仅供本人自己参考！并不具有强制意义！

我的编程规范/习惯

一、功能方面：

1. 标识符命名(变量、函数等)

  • 一律采用大小写分割单词，而不是下划线；
  • 局部变量名和函数名：
    除第一个单词之外其余单词首字母大写，如：
    局部变量名 userName
    函数名 getUserName；
  • 类中的成员变量名：
    每个单词首字母均大写，并在最前面加一个小写 m，如 mListView；

2. 函数返回值(状态，错误代码等)

  • 如果只是一个独立的过程调用，如输出帮助信息，则不必设置返回值，将返回值类型设为 void；
  • 除上一条之外，如果没有需要返回的有意义的参数，则默认返回函数运行状态，即，函数执行成功返回0，不成功返回1；

3. main函数输入参数处理

  • 操作选项：
    除第一个单词之外首字母大写，如果操作选项后有相应的参数，则在操作选项前面加一个 /，如 /r；
  • 在每个操作选项后可能会有数值参数；
  • 在main函数中判断输入命令时，将所有可能的输入参数事先写在一个数组里，方便判断逻辑；

4. 动态内存分配与释放

  • 每次释放内存之后紧跟着要将对应指针置为 NULL；

5. 控制台输出

  • 关于命令行参数的使用说明等信息单独写成一个函数；

二、美观方面：

1. 大括号 { }

  • 所有用于语句块控制的大括号都置于单独一行；
  • if、for、while 等语句的执行语句部分无论多少都要加大括号；
  • 定义数组的大括号，如果是一维数组且数组元素不多，则 { 和 } 放在同一行，否则，{ 和等号放在同一行，} 放在单独一行，且与 { 对齐；

2. 注释

  • 每个函数体前面要有一行 // 开头的注释，说明该函数的作用，输入输出参数的意义等；
  • 语句块注释：
    紧挨着放在语句块的上面，单独一行，用 //开头；
  • 单句的注释：
    放在需注释的代码后面，用 // 开头，// 前面加一个空格。如需对齐，则插入用空格代替的制表符；

3. 空行

  • 两个函数体之间要有两个空行；
  • 每个有独立意义或功能的语句块，上下要有一个空行，且最好在上面有一行注释；
  • 变量说明之后要加一个空行；

4. 缩进与对齐

  • 语句前面的缩进可以用制表符(Tab)，或者在编辑器中设置成将 Tab 自动转换为4个空格，暂且规定使用后者；
  • 行后的注释如需对齐，只能用上面的后者，或者手打空格；

2. 巧用数组0号元素

记录时间：2014-08-18
参考：严蔚敏，吴伟民，《数据结构(C语言版)》，清华大学出版社，P216，272。

数组的第0个位置可用作哨兵，或临时存储单元！可以在循环中利用！

3. 冒泡排序结束条件

记录时间：2014-08-18
参考：严蔚敏，吴伟民，《数据结构(C语言版)》，清华大学出版社，P273。

判别冒泡排序法结束的条件应该是“在一趟排序过程中没有进行过交换记录的操作”！以前没有注意到这一点，一直以为一定要进行 n-1 趟排序。

4. 字符串数组结束标志

记录时间：2014-08-23
参考：周纯杰，《标准C语言程序设计及应用》，P161，P165。

字符串数组最后可以加一个空字符串，作为循环结束的标志使用。
