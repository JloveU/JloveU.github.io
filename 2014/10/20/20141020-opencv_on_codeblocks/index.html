本文介绍了在CodeBlocks开发环境中使用OpenCV-2.4.9建立工程的方法，包括使用CMake编译OpenCV的过程。

背景

本人之前在C/C++开发过程中，使用的集成开发环境（IDE）一直是Visual Studio系列，其中用的时间最长的是VS2010，并且为了不使电脑运行变慢，一直在VMware虚拟机中安装VS软件。

最近又在虚拟机中新安装了VS2013 Professional with Update 3，安装过程用了将近一天的时间。其体积之大和安装速度之慢让我有了更换IDE的想法。于是，经过上网搜索，最终决定试着转向CodeBlocks。

由于我的专业是图像处理相关的，所以OpenCV用得比较多，之前在VS2010中使用OpenCV，只需要将官网下载的文件解压即可。而使用CodeBlocks建立基于OpenCV的工程的时候出了问题，在OpenCV-2.4.7及以后版本中没有了对应MinGW的库文件，即使是OpenCV-2.4.6及以前的版本中对应MinGW的库文件也需要和电脑上安装的MinGW/GCC编译器版本相同，否则编译后的程序会运行出错。于是，需要自己动手重新编译OpenCV。

CodeBlocks简介

Code::Blocks 是一个开放源码的全功能的跨平台C/C++集成开发环境。 Code::Blocks是开放源码软件。Code::Blocks由纯粹的C++语言开发完成，它使用了著名的图形界面库wxWidgets(2.6.2 unicode)版。对于追求完美的C++程序员，再也不必忍受Eclipse的缓慢，再也不必忍受VS.NET的庞大和高昂的价格。

（来自：Code::Blocks_百度百科）

MinGW简介

MinGW，是Minimalist GNU on Windows的缩写。它是一个可自由使用和自由发布的Windows特定头文件和使用GNU工具集导入库的集合，允许你在GNU/Linux和Windows平台生成本地的Windows程序而不需要第三方C运行时库。

（来自：mingw_百度百科）

OpenCV简介

OpenCV的全称是：Open Source Computer Vision Library。OpenCV是一个基于（开源）发行的跨平台计算机视觉库，可以运行在Linux、Windows和Mac OS操作系统上。它轻量级而且高效——由一系列 C 函数和少量 C++ 类构成，同时提供了Python、Ruby、MATLAB等语言的接口，实现了图像处理和计算机视觉方面的很多通用算法。

（来自：opencv_百度百科）

CMake简介

CMake是一个跨平台的安装（编译）工具，可以用简单的语句来描述所有平台的安装(编译过程)。他能够输出各种各样的makefile或者project文件，能测试编译器所支持的C++特性,类似UNIX下的automake。只是 CMake 的组态档取名为 CmakeLists.txt。Cmake 并不直接建构出最终的软件，而是产生标准的建构档（如 Unix 的 Makefile 或 Windows Visual C++ 的 projects/workspaces），然后再依一般的建构方式使用。这使得熟悉某个集成开发环境（IDE）的开发者可以用标准的方式建构他的软件，这种可以使用各平台的原生建构系统的能力是 CMake 和 SCons 等其他类似系统的区别之处。

（来自：cmake_百度百科）

很明显，OpenCV-2.4.7及以后版本中没有了mingw这个文件夹，也就是没有了对应MinGW编译器的库文件，因此需要自己动手编译生成。

  1. 到CMake官网下载CMake安装文件。
    目前最新版本是3.0.2，我下载的是2.8.12.2版本的Windows平台免安装文件，下载地址如下：
    http://www.cmake.org/files/v2.8/cmake-2.8.12.2-win32-x86.zip
  2. 将下载得到的 cmake-2.8.12.2-win32-x86.zip 文件解压到任意路径（可能需要路径中不包含中文），进入bin目录下，运行 cmake-gui.exe。
    （注：我一开始在Sandboxie沙箱中运行的这个程序，出现了很多问题，一直没有成功，后来不在沙箱中运行而是直接双击运行就成功了。）
  3. 点击 Browse Source...，选择解压出来的OpenCV中的source目录（OpenCV-2.4.6及以前版本选择最外层的opencv目录，也就是最外层的并且包含一个 CMakeLists.txt 文件的目录）；
    点击 Browse Build...，选择用来存放编译得到的库文件所在的文件夹（可以自己新建）。如下所示：
  4. 点击 Configure，如果上一步选择的Build文件夹不存在，将提示是否创建：

    点击 Yes 创建该文件夹。
  5. 在 Specify the generator for this project 处选择 CodeBlocks - MinGW Makefiles，下面选择 Specify native compilers。
  6. 点击 Next，选择编译器位置。

    因为我安装的CodeBlocks自带了MinGW/GCC编译器，CodeBlocks的安装目录为 D:\Program Files\CodeBlocks，其中包含MinGW的目录 D:\Program Files\CodeBlocks\MinGW。
    所以此处选择C编译器的位置为 D:/Program Files/CodeBlocks/MinGW/bin/gcc.exe，C++编译器的位置为 D:/Program Files/CodeBlocks/MinGW/bin/g++.exe。
  7. 点击 Finish。

    等待进度条结束（这个过程大概七八分钟，如果时间很长还没结束且停在一个地方，可能就失败了，我是在Sandboxie沙箱中运行时出现过这种情况）。
    当输出 Configuring done 时即代表完成此步骤。此时界面如下：
  8. 在红色区域下拉找到 CMAKE_BUILD_TYPE，在其值处输入 Debug，即生成Debug版的库文件，若输入 Release 或者留空，则生成Release版的库文件。

    再次点击 Configure。

    等待进度条结束（这个过程很短，大概几秒就结束）。
    当输出 Configuring done 时即代表完成此步骤。此时界面如下：
  9. 点击 Generate。

    等待进度条结束（这个过程也很短，大概十几秒就结束）。
    当输出 Generating done 时即代表完成此步骤。此时界面如下：
  10. 此时在Windows资源管理器中打开第3步中选择的存放编译得到的库文件所在的文件夹，其中的文件如下：

    其中有一个 OpenCV.cbp 文件，这是CodeBlocks的工程文件。
  11. 用CodeBlocks打开上面的 OpenCV.cbp 文件，即打开了用来编译OpenCV的工程。

    点击图中红色方框所示的按钮（Build），开始编译。
    Build log窗口会显示当前编译进度：

    此过程较长，需要一个多小时的时间。
  12. 等待编译结束。
    当出现以下结果时代表编译完成：
  13. 编译完成后，在Windows资源管理器中打开第3步中选择的存放编译得到的库文件所在的文件夹，其中有两个文件夹分别是 bin 和 lib，这两个文件夹就是以后基于OpenCV要用到的库文件。在第8步可以修改 CMAKE_BUILD_TYPE 的值，得到Debug和Release两个版本的库文件。
    将 bin 和 lib 这两个文件夹复制或剪切到OpenCV-2.4.9解压出来的目录 opencv/build/x86/mingw 下（mingw是自己新建的文件夹，代表支持MinGW/GCC编译器的库文件夹）。
  14. OpenCV编译完成！

在CodeBlocks中配置OpenCV环境

编译完OpenCV后，即可在CodeBlocks中创建基于OpenCV的工程，但需先配置OpenCV环境。

  1. 打开CodeBlocks，点击菜单 Settings->Global variables...。
  2. 点击下图中的New，新建全局变量。


    输入变量名：cv，点击OK。
  3. 设置cv变量的 base、include、lib、bin 的路径。

  4. 点击close，完成配置。

在CodeBlocks中新建基于OpenCV的工程

以上各步骤完成之后就可以新建基于OpenCV的工程了。

  1. 打开CodeBlocks，点击菜单 File->New->Project...。
  2. 在新建项目向导中，Category 处选择 Console，再点击 OpenCV project，然后点击go。
  3. 出现下面的提示界面，如果不希望以后出现此界面，则勾选 Skip this page next time，点击Next。
  4. 输入工程名，点击Next。
  5. 设置OpenCV的位置，此处就和配置OpenCV环境时设置的全局变量 cv 的 base 路径相同即可。点击Next。
  6. 设置OpenCV版本。例如我使用的OpenCV是2.4.9版本，于是将下面的 major、minor、revision 分别设置为 2、4、9。点击Next。
  7. 选择编译器。选择默认的 GNU GCC Compiler 即可。Debug和Release可按需要选择。
  8. 点击Finish，即完成了基于OpenCV的工程的创建。
